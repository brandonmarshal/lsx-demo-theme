### What went well?

-   **Productivity:** Completed all service pages planned for the week, working through Design, WordPress Dev, Performance & Accessibility, Support, Security, and Hosting pages
-   **Desktop & Mobile Menus:** Successfully created and connected all mega menus for desktop and mobile dropdowns in the Figma prototype
-   **WordPress Learning Curve:** The WordPress environment felt intuitive due to similarities with Figma's structure (groups, columns, rows, grids)
-   **Pattern Recognition:** Understanding improved about when and how to use patterns versus components, especially for maintaining consistency across menus
-   **Collaboration:** Work sessions with Zared and Ash were highly productive, helping to unblock issues and clarify design direction
-   **Table of Contents Implementation:** Successfully implemented a comprehensive table of contents for the prototype linking all pages and device sizes
-   **Responsive Design:** Efficiently created Tablet and Mobile versions of service pages, though some patterns required adjustments

### What have you learned?

-   **WordPress-Figma Parallels:** The setup in WordPress is very similar to Figma with grouping, columns, rows, etc., which made the learning curve smoother
-   **Pattern vs Component Usage:** Discovered that text blocks need to be patterns (not just components) for style changes to apply globally across all instances
-   **Workflow Clarity:** Learned the importance of having clear content requirements and design direction before starting page designs
-   **New Components:** Gained experience with 3D components and various new design elements that weren't previously in the design system
-   **Service Page Template:** Learned to apply a consistent page template across all service pages using content from planning documents
-   **Prototype Navigation:** Worked through the challenges of prototype navigation animations and got them as close to working as possible
-   **Hero Section Limitations:** Discovered that some hero sections use images where color tokens can't be applied, requiring design decisions about dark/light mode consistency

### Challenges Encountered

-   **Content Clarity (Days 3-5):** Initially unclear about how to style certain sections from the planning document - needed clarification on whether to use existing components, create new ones, or use cards. This was resolved in the Day 5 work session with Zared
-   **Pattern Consistency Issues (Day 2):** Text style changes weren't applying globally across all menus because text blocks weren't set up as patterns, causing repetitive editing
-   **New Components (Days 2-3):** Encountered 3D components and other elements not previously worked with, requiring time to understand and implement
-   **Broken Patterns (Day 3):** The services/design page had broken patterns that required creating custom designs for some sections, particularly affecting tablet and mobile views
-   **Hero Section Color Tokens (Day 4):** Hero sections on service pages use fixed images that can't have color adjustments, creating a potential issue with dark mode consistency
-   **AI Tool Limitations (Day 5):** Attempted to use Stitch to generate page layouts with new content from the document, but it wasn't working effectively
-   **Prototype Navigation Animations (Day 1):** Navigation animations were "a bit funky" and required workarounds to get as close to working as possible
